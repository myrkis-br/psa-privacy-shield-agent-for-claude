# PSA v6.0 — Risk Engine
Data: 2026-03-11
Autor: Marcos Cruz

## Resumo
Implementação do Risk Engine completo — classificação
automática de risco LGPD, enriquecimento de padrões via
cloud (sem dados reais) e relatório RIPD automático.

## Módulos implementados

### Módulo 1 — classifier.py
- classify_document() → risk_score 1-10
- Alinhado à Resolução ANPD nº 4/2023
- Detecta: tipo, subtipo, titulares, categorias sensíveis
- Classificação: leve / média / grave
- Falsos positivos corrigidos: biometria, racial, atividade

### Módulo 2 — pattern_enricher.py
- Privacy by Design: ZERO dado real transmitido
- Envia apenas metadados (tipo_doc + nomes de campos)
- Cache persistente: docs/patterns_learned.json
- Filtro de confiança: threshold 0.7
- Aprende padrões permanentemente
- Fallback graceful sem API key

### Módulo 3 — ripd_report.py
- Relatório RIPD automático (Art. 38 LGPD)
- Economia de tokens calculada (% e R$)
- Multa LGPD potencial evitada (Art. 52)
- Salvo em docs/ripd/RIPD_{DOC}_{data}.txt

### Módulo 4 — psa.py integrado
- Fluxo v6.0 completo
- Ajuste automático de cobertura por risk_score:
  score 1-3 → 30% | score 4-6 → 60% | score 7+ → 100%
- Seleção de modo quando custo >= R$ 0,10:
  ECO (30%) | PADRÃO (60%) | MÁXIMO (100%)
- Log de responsabilidade com timestamp
- Aviso obrigatório: GRAVE + ECO requer confirmação
- Retrocompatível: --register, --list-files, --history OK

detectava 42 entidades. Com cobertura 100% (risk 9):
85 entidades — 43 entidades que estavam escapando.

A cloud recebe apenas metadados para aprender padrões."
